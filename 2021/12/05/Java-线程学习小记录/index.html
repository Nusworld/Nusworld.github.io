  1. 传递中断 在Interrupt线程时，线程抛出异常，但是在处理异常时应该在线程的顶层方法中处理异常即进行try catch, 而在较低的层次不要使用try catch来“吞掉”异常，应该使用函数签名处理，即 throws 相关Exception

  3. interruped() 该静态方法只关心运行它的线程，在返回线程状态后，重新设置标志位为false 而isInterrupted() 方法不是静态方法，返回的是对应线程实例的状体，且不会改变标志位

        运行结果可能存在的情况

        结果 结果描述
        b = 3 a = 3 线程一先全部执行完，线程二后执行完
        b = 2 a = 1 线程二先全部执行完，线程一后执行完
        b = 2 a = 3 线程一执行完a的修改，后线程二执行
        b = 3 a = 1 可见性问题发生，线程二只能看见线程一的部分修改

        问题解决：使用volatile关键字

        volatile int a = 1;
volatile int b = 2;
